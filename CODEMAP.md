## Architecture

`codetalk-cli` is a command-line tool (package name `codetalk-cli`, version 0.1.3) for analyzing codebases, producing living semantic maps, and executing safe, planned changes via LLM agents. It is written in TypeScript and compiled to `dist/index.js` (two binaries: `codetalk` and `codetalk-cli`).

**Main execution path**: The user invokes a CLI command (`help`, `init`, `config`, `scan`, `map`, `ask`, `plan`, `exec`, `check`, `rollback`). The entry point (`src/index.ts`) parses arguments and dispatches to a handler in `src/handlers.ts`. Each handler orchestrates the required logic: config loading via `src/utils.ts`, LLM API calls via `src/api.ts`, tool execution via `src/tools/index.ts`, symbol indexing via `src/indexer.ts`, and progress display via `src/panel.ts`. The file walkers now honor `.gitignore` so scans, indexes, and code search stay aligned with Git's ignore rules. The LLM agent (OpenAI-compatible) receives system prompts from `src/prompts.ts` and tool definitions, and may call back into the tool layer.

**Major components**:
- **CLI dispatcher** (`src/index.ts` + `src/constants.ts`): argument parsing, help generation, command routing.
- **Handlers** (`src/handlers.ts`): implementation of all commands, plus sub‑orchestrators for scanning, planning, execution, and semantic map synchronization.
- **LLM API client** (`src/api.ts`): non-streaming and streaming chat completions, token usage tracking, tool call handling, panel progress.
- **Tool system** (`src/tools/`): read, grep, ls, glob, stat, git_log, search – invoked by the agent during planning/execution.
- **AST extractors** (`src/ast/`): extract symbols from TypeScript, Python, C++, and assembly files.
- **Indexer** (`src/indexer.ts`): builds in‑memory symbol indices from source trees, used by `search` and `grep` tools.
- **Prompt templates** (`src/prompts.ts`): system prompts, tool definitions, and prompts for ask, plan, semantic sync, reviewer agents, etc.
- **Config** (`src/utils.ts`): read/write JSON config file (`~/.codetalker/config.json`), manage API URL, key, model.
- **Semantic map generator** (`src/utils.ts`): produces `CODEMAP.md` from repository structure (`buildMap`, `buildTemplate`, `buildChangeSync`).

**Scale**: 40 source files (~150 KB total). Primary language: TypeScript (25 files). Designed for single-user CLI use, no server.

**Design philosophy**: Agent‑centric; the semantic map (`CODEMAP.md`) is the behavioral contract for LLM agents. Change safety via explicit protocol (plan → execute → sync). Rollback support via backups.

## Runtime Flow

### Startup
1. User invokes `codetalk <command>` (or `codetalk-cli`).
2. `dist/index.js` executes; `src/index.ts` parses `process.argv` using `src/constants.ts` for command definitions.
3. If no valid command, prints help and exits.
4. Config is loaded via `src/utils.ts::readConfig()` (or interactively in `config` command).
5. Command handler function is called with parsed options.

### Normal Execution
- **`init`**: Writes template `CODEMAP.md` to current directory.
- **`config`**: Interactive prompts for API URL, key, model; writes config file.
- **`scan`**: `scanRepo()` → `collectSourceFiles()` → `summarize()` → `buildScanReport()` → `writeMap()`. Optionally runs deep analysis via `runArchitectureScan()`.
- **`map`**: Generates baseline `CODEMAP.md` from repository structure using `buildMap()`.
- **`ask`**: `askCodebase(question)` → reads current map and repository evidence → calls `runPrompt()` or streaming variant → displays answer.
- **`plan`**: `planChange(request)` → uses LLM with tools to generate `CODEPLAN.md` (YAML).
- **`exec`**: `execution()` → reads `CODEPLAN.md` → parses change specs → creates backup → spawns parallel editor agents (configurable concurrency) → applies changes → auto‑syncs semantic map.
- **`check`**: Verifies `CODEMAP.md` exists and is up‑to‑date with source files; fails if not.
- **`rollback --list`**: Lists backup IDs. `rollback <id>`: restores files from backup.

### Error Paths
- Config missing: many commands prompt or fail with clear message.
- LLM API unreachable: `src/api.ts` catches network errors, may retry (depending on caller).
- Tool execution failure (e.g., file not found, grep syntax error): returns error `ToolResult` with `error: true`.
- Invalid user input: CLIs typically print usage and exit with code 1.
- Plan execution failure: retry up to a limited number of times (via `retryEditorPrompt`); if still fails, mark as failed and continue.

### Teardown
- No explicit cleanup; process exits after handler returns. Terminal state restored by `MissionPanel` (cancels spinner).
- Backups stored in `.codetalk-backups/` directory (not cleaned automatically).
