## Architecture

- **Purpose**: `codetalk-cli` is a command-line tool for generating, maintaining, and executing semantic maps over a codebase. It scans repository structure, analyzes code via LLM reviewers, produces a `CODEMAP.md` (semantic contract), generates safe implementation plans (`CODEPLAN.md`), and executes those plans against the source tree, auto-syncing the semantic map.
- **Main execution path**:
  1. `npx codetalk init` – creates a template `CODEMAP.md`.
  2. `npx codetalk scan` – uses parallel LLM reviewers to inspect source files and produce architecture semantics.
  3. `npx codetalk map` – generates a baseline semantic map from repository structure (file tree, language detection, entry candidates).
  4. `npx codetalk plan "request"` – reads the semantic map and source files, then writes an implementation plan to `CODEPLAN.md`.
  5. `npx codetalk exec` – applies the plan’s file changes in parallel via LLM calls and updates `CODEMAP.md` in the same transaction.
  6. `npx codetalk ask "question"` – answers codebase questions using map and scan context.
  7. `npx codetalk check` – fails if `CODEMAP.md` is missing or stale (newer source files).
  8. `npx codetalk rollback` – restores files from a previous exec backup.
- **Major components**:
  - **CLI entry point** (`src/index.ts`) – defines commands and routes to handlers.
  - **Handlers** (`src/handlers.ts`, 44 KB) – core orchestration logic for each command (init, scan, map, plan, exec, ask, check, rollback, config).
  - **API** (`src/api.ts`, 17 KB) – implements communication with remote LLM services (likely MCP/OpenAI-compatible).
  - **Indexer** (`src/indexer.ts`, 7.8 KB) – scans file system (glob, grep, git log) and extracts repository structure.
  - **Prompts** (`src/prompts.ts`, 24 KB) – stores and manages LLM prompt templates for each agent role.
  - **Utilities** (`src/utils.ts`, 27 KB) – shared helper functions (formatting, file I/O, error handling).
  - **Tools** (`src/tools/`) – a set of file system tools (glob, grep, ls, read, search, stat, git_log, shared, types) that wrap system commands and provide structured results.
  - **Panel** (`src/panel.ts`, 2.6 KB) – likely an interactive panel/UI for the chat interface (may use `ink` or similar).
  - **Constants** (`src/constants.ts`, 9.6 KB) – configuration values, default model names, API URL defaults, color codes.
  - **Types** (`src/types.ts`, 1.6 KB) – shared TypeScript type definitions.
  - **Test harness** (`scripts/test-cli.mjs`, 7.0 KB) – dependency-free smoke test exercising CLI commands interactively.
- **Scale**: Single‑process CLI tool; operated from terminal; state is kept in the local repository (files and one config file at `~/.codetalker/config.json`).
- **Design philosophy**: Semantic maps as contracts; LLM‑driven analysis and code generation; idempotent operations; parallel LLM calls for speed; plan-then-execute workflow with rollback safety.

## Runtime Flow

### Startup
1. User runs `codetalk <command> [args]` from terminal.
2. `src/index.ts` parses arguments using `yargs` or equivalent.
3. If command requires a config, loads `~/.codetalker/config.json` (if missing, prompts user via `codetalk config`).
4. If command requires a semantic map (`map`, `plan`, `exec`, `ask`, `check`), verifies that `CODEMAP.md` exists and is not stale (compare timestamps of source files vs map). If stale or missing, fails with error (unless `--force` or `init` command).
5. Dispatches to the appropriate handler function in `src/handlers.ts`.

### Normal execution paths
- **`codetalk init`**: Handler writes a template `CODEMAP.md` with skeleton sections. Exits 0.
- **`codetalk scan [--stream] [--parallel N]`**: Handler calls `src/indexer.ts` to get file list, then spawns parallel LLM reviews via `src/api.ts`; results are aggregated and written to stdout or a file.
- **`codetalk map`**: Uses `src/indexer.ts` to build module table, entry candidates, language detection; reads existing `CODEMAP.md` if present; merges with current state; writes updated `CODEMAP.md`.
- **`codetalk plan "request"`**: Reads `CODEMAP.md` and relevant source files; forms a prompt using templates from `src/prompts.ts`; sends to LLM via `src/api.ts`; parses response into `Plan`; writes `CODEPLAN.md`.
- **`codetalk exec`**: Reads `CODEPLAN.md`; creates a backup snapshot (copy current files to `~/.codetalker/backups/`); for each step, calls LLM to generate new content (parallel); applies changes to source files; updates `CODEMAP.md` with any semantic changes; logs changes.
- **`codetalk ask "question"`**: Reads `CODEMAP.md` and optionally scan results; forms prompt; streams LLM answer to stdout.
- **`codetalk config`**: Interactive dialog reading/writing `~/.codetalker/config.json`.
- **`codetalk check`**: Compares modification timestamps of all source files (`.ts`, `.mjs`, etc.) against `CODEMAP.md`’s timestamp; exits 0 if map is newer, 1 otherwise.
- **`codetalk rollback [id]`**: Copies files from backup directory back to project root.

### Error paths
- **Missing config**: Config‑dependent commands ask user to run `codetalk config` first.
- **Stale or missing `CODEMAP.md`**: `codetalk plan`, `exec`, `ask`, `check` fail with descriptive error.
- **LLM API failure**: Retries up to N times (configurable); on final failure, logs error and exits 1.
- **File write failure** (permission, disk full): Backups may remain; user can rollback.
- **Invalid plan syntax**: Plan parsing fails; exec stops before applying changes.
- **Git not found** (for `git_log` tool): Returns empty result or error.

### Teardown
- CLI always exits with appropriate exit code.
- No persistent background processes.
- Temporary files (backup snapshots) remain on disk until manually cleaned or overwritten.
